EL表达式与JSTL

1.EL表达式

1.1概述

EL全称为Expression Language，即表达式语言。它是JSP 2.0引入的一个新内容。EL可以简化JSP开发中的对象引用，从而规范页面代码，增加程序的可读性及可维护性。EL为不熟悉Java语言页面开发的人员提供了一个开发JavaWeb应用的新途径。

1.2语法

EL的语法非常简单，都是以“${”符号开始，以“}”符号结束的，具体格式如下:

${表达式}

1.3举例

//使用EL表达式绑定的数据
        req.setAttribute("abc","guo");
        req.setAttribute("def","li");
//开启el表达式        
        <%@ page isELIgnored="false" %>
<%--方式一，通过原来方式--%>
       <%=(String) request.getAttribute("abc")%>
       <%=(String) request.getAttribute("def")%>
<%--方式二，使用el表达式获取，需要在网页头部开启el表达式--%>
       <hr>
       ${abc}
       ${def}

使用EL同样可以成功获取Servlet中存储的数据，但EL明显简化了JSP页面的书写，使程序简洁易维护。另外，当域对象里面的值不存在时，使用EL获取域对象里面的值时返回空字符串；而使用==Java方式获取时，如果返回值是null，会报空指针异常==，所以在实际开发中推荐使用EL的方式获取域对象中存储的数据。

1.4 命名规范

在EL中，经常需要使用一些符号标记一些名称，如变量名、自定义函数名等，这些符号被称为标识符。EL中的标识符可以由任意的大小写字母、数字和下划线组成，为了避免出现非法的标识符，在定义标识符时还需要遵循以下规范：

（1）不能以数字开头。

（2）不能是EL中的关键字，如and、or、gt等。

（3）不能是EL隐式对象，如pageContext。

（4）不能包含单引号（’）、双引号（”）、减号（-）和正斜线（/）等特殊字符。

2.el表达式的作用

<%@ taglib prefix="c" uri ="http://java.sun.com/jsp/jstl/core"%>
//替换项目中中的代码
    <c:forEach var="u" items="${users}">
            <p>
            ${u.id} ${u.code} ${u.password}
            <a href="del?id=${u.id}"> 删除</a>
            <a href="toUpda?id=${u.id}" >修改</a>
            </p>
     </c:forEach>

     <form action='upda' method='post'>
            <p>id:<input type='text' name='id' readonly value='${u.id}'></p>
            <p>账号:<input type='text' name='zh' value='${u.code}'></p>
            <p>密码:<input type='password' name='mm' value='${u.password}'></p>
            <p><input type='submit' value='修改'></p>
            <p><input type='reset' value='重置'></p>
        </form>

2.1访问javabean 的属性

通常情况下的JavaBean指的是一个公共的类，含有一个空参的构造方法, 一些属性以及访问这些属性提供get/set方法,方法名与属性名需要符合一定的规范。使用EL表达式访问Bean属性时可使用如下两种方式
-方式一: ${对象名.属性名}

-方式二: ${对象名[“属性名” ]}

2.2输出简单的运行结果

EL表达式可以做一些简单的计算，可将结果直接输出或给JSP标签的属性赋值

算术运算： + - * / % 注意：+号只能求和，不能用于连接字符串

逻辑运算：&& || ！

关系运算： >= > <= < == !=

empty :用来判断-个字符串是否为空,或者-个集合是否为空，以下四种情况结果为true :空字符串，空的集合,值为null ,找不到对应的值

2.3获取请求参数值

$ { param . username }
等价于
request.getParameter ( "username” ) ;
$ { paramValues . city}
等价于
request.getParameterValues( "city" ) ;

3.JSTL

3.1概述

JSP就支持使用自定义标签，使用自定义标签大大降低了JSP页面的复杂度，同时增强了代码的重用性。

JSTL : ( JSP Standard Tag Library )JSP标准标签库JSP标签是Sun公司定义的一套标准，由Apache组织基于这套标准开发的一套标签库后又转给Sun公司，被称为JSTL，成为了JavaEE5.0的核心

3.2 引入

<dependency>
  <groupId>Jstl</groupId>
  <artifactId>Jstl</artifactId>
  <version>1.2</version>
</dependency>
使用taglib指令导入要使用的JSP标签
< %@taglib uri="" prefix=""%>
uri : JSP标签的命名空间
prefix :命名空间的前缀

3.3常用标签库

使用核心标签库的taglib指令格式如下:
<%@ taglib prefix=”c” url= http:/java.sun.com/jsp/jst/core” %>
核心标签库主要用于完成JSP页面的常用功能，包括JSTL的表达式标签、 URL标签、 流程控制标签。
例如，用于输出文本内容的<c:out>标签、用于条件判断的<c:if> 标签、用于迭代循环的
<c:forEach>标签等。
使用格式标签库的taglib指令格式如下:
<%@ taglib prefix “fmt” ut=ttp://java.sun.com/jsp/jt/fmt”%*>
国际化/格式化标签库，它包含实现Web应用程序的国际化标签和格式化标签。例如，设置JSP页面
的本地信息、设置JSP页面的时区、 使日期按照本地格式显示等。
使用XML标签库的taglib指令格式如下:
<%@ taglib prefx=’ “xml” url= http://java.sun.com/jsp/jstl/xml" %>
XML文档的标签库，它包含对XML文档中的数据进行操作的标签。例如，解析XML文件、输出XML
文档中的内容，以及迭代处理XML文档中的元素。XML广泛应用于Web开发,使用XML标签库处理
XML文档更加简单方便。
使用函数标签库的taglib指令格式如下:
<%@ taglib prefix= “fn” url= “http://java.sun.com/jsp/jst/functions"%>
函数标签库，它提供了一套自定义EL函数,包含了JSP网页制作者经常要用到的字符串操作。例如,
提取字符串中的子字符串、获取字符串的长度等。

3.4 标签库使用

  1. 用来遍历集合或者数组
    语法:
    <c: forEach var= “”items= “”; >
    </c: forEach>
    items属性 :指定要遍历的集合 ,-般使用EL表达式来复制
    var属性 :指定一个绑定名称,容器每次从集合中取一个对象,然后绑定到pageContext对象上
    varStatus属性 :指定一个绑定名称,绑定值是一个由容器创建的对象,该对象封装了当前迭代的状态
  2. 语法: ... when表示一 个处理分支,当test属性为true时会执行该分支,可以出现1次或者多次 otherwise表示例外,可以出现0次或1次
  3. 语法:
    <c:if test =“”var=
    scope=””>

    当test属性值为true时,执行标签体的内容。test属性可以使用EL表达式赋值
    var属性:指定一个绑定名称
    scope属性:指定绑定的范围( page , request,session , application )
    注: var和scope要配合使用
